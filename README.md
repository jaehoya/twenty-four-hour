# ☁️ Twenty Four Hour

24학번 팀원들이 뭉쳐 24시간 열심히 개발 공부하자는 의미를 담은 클라우드 기반 파일 저장 및 공유 서비스 프로젝트입니다.

사용자에게는 언제 어디서나 접근 가능한 편리한 파일 저장소를, 팀원들에게는 **실무 중심의 협업 경험**을 제공하는 것을 목표로 합니다.

## 🚀 주요 기능

프로젝트는 3단계로 나누어 점진적으로 기능을 구현하며 개발하였습니다.

### 1️⃣ 1차 개발 (핵심 기능) *개발 완료*

서비스의 근간이 되는 핵심 기능입니다.

* **인증 (Authentication):** JWT(JSON Web Token)를 활용한 안전한 로그인, 회원가입, 비밀번호 찾기/변경 기능을 제공합니다.
* **메인 대시보드 (Dashboard):** 사용자의 모든 파일과 폴더를 한눈에 볼 수 있는 직관적인 UI를 제공하며, 실시간 검색 및 이름/날짜순 정렬 기능을 지원합니다.
* **파일/폴더 관리 (File Management):**
  * 신속한 파일 업로드 (Drag & Drop 인터페이스 포함)
  * 필요한 파일의 즉각적인 다운로드
  * 파일/폴더 이름 변경, 삭제, 즐겨찾기 등 핵심 관리 기능
* **UI/UX:**
  * 대용량 파일 업로드 시 사용자가 인지할 수 있는 진행률 표시
  * 모든 API 요청(공, 실패, 오류)에 대한 명확한 피드백 메시지
  * 현재 저장 용량과 최대 용량을 시각적으로 보여주는 게이지 바
* **프로필 (Profile):** 사용자 프로필 사진을 자유롭게 추가하고 변경할 수 있습니다.

### 2️⃣ 2차 개발 (선택 기능) *진행중*

사용자 경험을 향상시키고 편의성을 높이는 기능입니다.

* **공유:** 파일 및 폴더에 대한 고유 공유 링크를 생성하고, 클립보드 복사 기능을 제공합니다.
* **미리보기:** 이미지 파일을 클릭했을 때 모달(Modal) 창 등에서 이미지를 바로 확인할 수 있습니다.
* **테마:** 사용자의 선호도에 따라 라이트 모드와 다크 모드를 선택할 수 있는 전환 토글을 제작합니다.

### 3️⃣ 후순위 (고급 기능) *추후 예정*

서비스의 완성도를 높이고 전문적인 기능을 추가합니다.

* **휴지통:** 삭제된 파일은 즉시 영구 삭제되지 않고 휴지통으로 이동하며, 사용자는 이를 복원하거나 영구 삭제할 수 있습니다. (Soft Delete 구현)
* **고급 미리보기:** 비디오 플레이어를 연동하여 스트리밍을 지원하고, HWP, Docs, PDF 등 주요 문서 파일의 내용을 보여주는 뷰어를 연동합니다.
* **구독:** 구독 플랜(무료/유료)에 따라 사용자별 저장 용량 및 접근 권한을 차등 관리하는 시스템을 도입합니다.

## 🤝 협업 및 관리

본 프로젝트는 체계적인 관리와 원활한 소통을 통해 협업 효율을 극대화합니다.

### Git Flow 전략

안정적인 코드 통합과 배포를 위해 Git Flow 전략을 채택합니다.

* **브랜치 분리:** `develop/frontend`와 `develop/backend` 브랜치를 분리하여 파트별 독립적인 개발.
* **PR(Pull Request) 규칙:**
  * 모든 코드는 `develop` 브랜치로 merge하기 전 PR을 통해 동료 리뷰(Code Review)를 거칩니다.
  * 커밋 메시지는 `[추가]`, `[수정]`, `[기록]` 등으로 변경 사항을 명시하여 히스토리 추적을 용이하게 합니다.

### 회의 및 커뮤니케이션

명확한 목표 공유와 신속한 이슈 해결을 위해 정기적인 소통 채널을 운영합니다.

* **정기 회의:** 격주 1회 (목요일) 전체 팀원이 참여하여 진행 상황을 공유하고, 다음 2주간의 개발 목표를 설정.
* **중간 점검:** 격주 1회 (목요일) 개발한 내용 간단 보고 및 이슈 공유.
* **커뮤니케이션:** Discord를 메인 채널로 사용하여 개발 이슈를 관리하고, 카카오톡을 통해 빠른 공지사항을 전달합니다.

## 📌 프로젝트 현황 (중간 발표 기준)

* **시연 흐름:**
  1. **회원가입:** 신규 사용자를 등록합니다.
  2. **로그인:** 인증 토큰을 발급받아 메인 페이지로 이동합니다.
  3. **파일 업로드:** Drag & Drop 또는 버튼 클릭으로 파일을 서버에 저장하고 목록에 표시합니다.
  4. **파일 다운로드:** 목록에서 파일을 선택하여 서버로부터 로컬 환경으로 다운로드합니다.
  5. **파일 삭제:** 파일을 선택하여 서버와 목록에서 제거합니다.
* **배포:** 현재 임시 배포중.
